# 20260312-Dashboard-Analytics-Enhancement

## 検討事項

### 1. 返信統計の算出
- **返信率**: `threads`データの中から、`replies`配列が空でないものの割合を算出。
- **返信数分布**: 各スレッドの返信数をカウントし、0件、1件、2件、3件、4件、5件以上のバケットで集計。
- **意図**: コミュニケーションの活発さを可視化することで、一過性の投稿か対話が発生しているかを判別可能にする。

### 2. 完了時間の比較
- **ロジック**: `completed_at` - `created_at` を計算。
- **比較軸**: 「返信あり」と「返信なし」のグループに分け、それぞれの平均値を算出。
- **仮説**: 返信が多いものは複雑な課題であり、完了まで時間がかかる可能性がある。逆に、返信がないものは即時解決されている。
- **意図**: 返信の有無が業務効率（解決スピード）に与える影響を分析する。

### 3. 長時間スレッドの抽出 (Top 100)
- **抽出条件**: ステータスが `completed` かつ、(完了時間 - 作成時間) が大きい順にソート。
- **表示内容**: タイトル、担当者（完了者）、所要時間。
- **意図**: ボトルネックとなっている業務や、対応に苦慮した事例を特定しやすくする。

## 実装方針
- 既存の `useThreads` フックですべてのスレッドデータ（リプライ込み）を取得できているため、フロントエンド側での集計処理のみで対応可能。DB負荷を抑えつつリアルタイムな分析を実現する。
