# 更新履歴 (CHANGELOG)

## [2026-03-26]
### 修正
- **アイコン配色とホバーエフェクト (Liquid Glass UI)**: 通常時は白、ホバー時に各アクション固有の色（水色・黄色・シアングリーン・紫）で「スケルトン状に光る」グローエフェクトを実装。直感的な操作感と高級感を両立。
- **SSOログイン不具合の解消**: メールアドレスの照合を case-insensitive (`ilike`) に変更。Microsoft 側と Supabase 側のメールアドレスの大文字小文字が異なる場合でもログイン可能に改善。
- **UIクリーンアップ**: 返信カードの枠線調整、メンションの二重枠解消、スレッド内の不要な境界線の削除を行い、よりクリーンなデザインを実現。

## [2026-03-24]
### 修正
- **Microsoft サインイン不具合の解消**: `redirectUri` の取得ロジックを `.env` 優先に変更し、Azure Portal 設定との不一致を解消。ログインガード（`isLoggingIn`）を強化し、多重ポップアップによる CORB エラーを防止。
- **リンクプレビュー（OGP取得）機能の再実装**: プロジェクトから欠落していた `get-link-preview` Supabase Edge Function を `deno-dom` を用いて再構築。
- **画像アップロード機能の品質検証**: `c1600x1600` サムネイル取得ロジックおよび即時プレビュー表示の整合性を最終確認。
 
## [2026-03-21]
### 追加
- **画像アップロード即時プレビューと詳細確認**: 
    - 投稿・返信時にファイルを選択または画像を貼り付けた際、アップロード完了を待たずに即座にプレビューを表示する機能を実装。
    - プレビューの表示サイズを拡大（最大 200x150）し、送信前に内容を確認しやすく改善。
    - プレビュー画像をクリックすることで、送信前にフルサイズで確認できる機能を追加。
- **画像表示品質の大幅向上 (800px制限の解除)**: 
    - OneDrive API から `c1600x1600`（1600px カスタムサイズ）のサムネイルを取得するように変更。
    - 従来の `large`（最大800px）によるボケを解消し、より高精細なプレビューに対応。
    - CSS `image-rendering: -webkit-optimize-contrast` を導入し、縮小表示時の鮮鋭度を向上。
## [2026-03-12]
### 追加
- **URL自動リンクとプレビュー機能**: フィード投稿および返信内のURLを自動的に検出し、クリック可能なリンクに変換。プレビューカードの表示に対応。
- **メンション候補の動的フィルタリング**: 投稿・返信時のメンション候補を、現在のチームに所属しているメンバーのみに制限。
- **メール連携ロジックの厳格化**: チームアドレスが「To」と「Bcc」の両方に含まれる場合のみ、スレッドを作成するように条件を強化。

### 修正
- **メールスレッド作成ロジックの改善**: `handle-inbound-email` において、メールアドレスの抽出精度を向上させ、ログ出力を詳細化。
- **ダッシュボード分析機能の強化**:
    - 返信率、総返信数、返信数分布の可視化。
    - 返信の有無による解決スピード（完了時間）の比較分析。
    - 解決まで時間を要した「長時間スレッドランキング (Top 100)」の追加。
    - ランキングの号機（スレッドタイトル）クリックで該当スレッドへ直接遷移するナビゲーション機能を実装。
- **UI/UXの改善**:
    - ランキングテーブルの表示崩れ（オーバーラップ）を修正。
    - リスト行のホバーエフェクトと強調表示を追加。
    - 各種アイコンのSVG化（編集・削除アクション）。

 ## [2026-03-04]
### 修正
- **PC画面におけるモバイルUI表示の不具合を修正**: PCブラウザ等の幅の広い画面（デスクトップ表示）において、モバイル専用の下部ナビゲーション（「フィード」「未完了」等）および投稿ボタンが表示されてしまう不具合を修正しました。`min-width: 769px`のメディアクエリを追加し、モバイル用要素をPC画面で確実に非表示に設定しました。

## [2026-02-24]
### 計画・設計
- モバイルアプリ化に向けた要件定義と仕様書 (`仕様書.md`) を作成。PWAとCapacitorの2つのアプローチを比較検討の末、既存Push通知との互換性を保つため「手動マニフェスト配置によるPWA化」に決定。

### 追加
- PWA（Progressive Web App）としてのインストールに対応。
  - `public/manifest.json` を追加し、`index.html` に関連メタタグを設定。モバイルブラウザから「ホーム画面に追加」を行うことで、ネイティブアプリのような全画面表示が可能になります。

### 修正
- **未完了タスク表示の不具合を修正**: 所属チームを持たないユーザーが「Not Finished」欄で全チームの未完了タスクを閲覧・完了できてしまうバグを修正。管理者以外でチーム選択がない場合は非表示となります。

- **メンション（@タグ）機能の復元**: 欠落していた `useTags` フックの呼び出しを復元し、グループメンションを再度有効化。

- **コードのクリーンアップ**: 未使用のインポートや変数に起因する lint 警告を多数解消。
